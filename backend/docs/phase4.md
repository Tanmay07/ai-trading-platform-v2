# Phase 4: AI Supervisor, Multi-Agent Consensus & Explainable AI

## Overview
Phase 4 replaces the rigid algorithmic weighting logic with a highly dynamic **Multi-Agent Consensus System**. Independent autonomous agents evaluate specific subsets of candidate features, adjust their influence dynamically based on the market regime, and achieve a consensus decision finalized by an AI Supervisor. 

## The 9 Autonomous Agents
All agents live in `backend/app/ai/` and inherit from `BaseAgent`. They ingest the rich data object built by Phases 1-3, and output a standardized `AgentResponse`:
1. **TechnicalAgent**: Evaluates EMA alignment, RSI, and ATR.
2. **BreakoutAgent**: Evaluates Phase 2 Breakout patterns and Resistance proximity.
3. **MomentumAgent**: Evaluates Relative Strength and MACD.
4. **VolumeAgent**: Evaluates Relative Volume Expansion and OBV trends.
5. **MarketAgent**: Evaluates the broader Phase 3 Market Score and Health.
6. **MacroAgent**: Evaluates the Phase 3 Macro Score and VIX.
7. **SentimentAgent**: Evaluates the Phase 1 FinBERT/VADER Sentiment.
8. **RiskAgent**: Evaluates Reward-to-Risk ratio and Stop Loss tightness.
9. **PortfolioAgent**: Evaluates Phase 1 Sector ranking.

## Key Sub-Engines
- **AdaptiveWeightEngine**: Reads `config/ai.yaml`. In a "Strong Bull" market, Technical/Momentum carry 55% weight. In a "Bear" market, Risk/Market/Macro carry 75% weight. 
- **ConsensusEngine**: Mathematically combines the 9 votes, resolving Agreement Percentages, weighted Confidences, and Conflict Scores.
- **SupervisorAgent**: Applies hard-overrides to the consensus. For example, if the Consensus is BUY, but Conflict is >30%, the Supervisor downgrades it to HOLD. If the Market is Bearish and the RiskAgent votes SELL, the Supervisor overrides the final output to SELL.
- **ExplainabilityEngine**: Synthesizes the individual `+` and `-` reasons generated by agents into a human-readable summary.

## Data Flow
The AI Orchestrator acts as an interceptor in the `RecommendationEngine`.
1. Features are built by Phase 1.
2. Technical Patterns are scored by Phase 2.
3. Global Context is built by Phase 3.
4. The candidate is passed to `AIOrchestrator.evaluate_candidate()` where all 9 agents execute concurrently via `asyncio.gather()`.
5. The Supervisor replaces the legacy `Confidence` metric with the AI Consensus Confidence.
6. The legacy Phase 1 Position Sizer proceeds to allocate capital based on this final verdict.

## API Endpoint
`GET /api/ai-recommendations`
Returns a fully transparent output for each recommended stock, exposing all individual agent scores, consensus agreements, adaptive weights, and human-readable summaries.

## Performance
Because all agents purely consume the pre-calculated dictionary of features generated by prior phases, there is zero added network latency. The `asyncio.gather()` evaluation layer typically adds less than 100ms.
